# Бот для подготовки к экзаменам (Telegram)

Бот-тренажёр по билетам с карточками, вопросами от ИИ и интервальным
повторением. Загружены два предмета:

- **Аналитическая химия** — 50 билетов
- **Фармацевтическая технология** — 31 билет

ИИ работает на **OpenAI API**.

## Что умеет

- **Выбор предмета** — у каждого своя колода, статистика и расписание повторений.
- **Карточки** — билет → «Показать ответ» → самооценка **Знал / Частично / Не знал**.
  На «Частично» и «Не знал» ИИ объясняет тему глубже.
- **Вопрос от ИИ** — модель задаёт вопрос по материалу, студентка отвечает
  текстом, ИИ проверяет и указывает на ошибку.
- **Интервальное повторение (SM-2)** — бот сам решает, что показать сегодня.
- **Статистика** — готовность в %, сильные/слабые билеты, список «повторить
  в первую очередь», streak, обратный отсчёт до экзамена.
- **Напоминания** — ежедневно в выбранный час.

Все кнопки встроены в сообщения.

## Главное: ИИ отвечает ТОЛЬКО по материалу билетов

Каждый запрос к модели (объяснение, генерация вопросов, проверка ответа)
получает эталонный текст билета и системную инструкцию отвечать строго по
нему, ничего не добавляя из общих знаний. Если чего-то нет в материале —
модель это проговаривает, а не выдумывает. Логика в `ai.py`.

## Установка

```bash
pip install -r requirements.txt
cp .env.example .env      # затем впиши BOT_TOKEN и OPENAI_API_KEY
python bot.py
```

- `BOT_TOKEN` — у @BotFather в Telegram (`/newbot`).
- `OPENAI_API_KEY` — на platform.openai.com/api-keys.
- `AI_MODEL` — модель OpenAI (имена на platform.openai.com/docs/models).
  `gpt-4o-mini` — дёшево и достаточно; `gpt-4o` — точнее.

База `bot.db` и билеты из `data/*.json` подхватываются при первом запуске.

## Обновить/добавить предмет из docx

Нужен `pandoc`. Пример для ФТ:

```bash
python parse_docx.py фт_шпоры.docx ft "Фармацевтическая технология" > data/ft.json
```

Перезапусти бота — импорт идемпотентный, повторный запуск обновляет билеты.

## Структура

```
bot.py          — точка входа, все обработчики, FSM, напоминания
config.py       — чтение .env
db.py           — SQLite: схема + выборки + статистика
srs.py          — интервальное повторение (SM-2)
ai.py           — запросы к OpenAI, привязанные к материалу билета
keyboards.py    — встроенные кнопки
parse_docx.py   — разбор docx-шпор в JSON билетов
data/           — analytical.json, ft.json
```

## Что стоит проверить глазами

Материал — реальные «шпоры» с рваной нумерацией. Парсер разобрал их
эвристикой (номер билета строго возрастает; плотные нумерованные списки
реакций/шагов отсекаются). Мелочи для проверки:

- **Аналитическая химия, билет №42** («Схема анализа смеси анионов первой
  группы») — без текста, в исходнике это схема-картинкой. Впиши текст в
  `data/analytical.json`.
- Часть формул во втором файле была картинками и в текст не попала — для
  таких билетов ответ ИИ будет неполным, стоит дозаполнить эталон.
- Пропущенные номера (аналитика: 8–11, 18–24 и др.; ФТ: 27) — билетов не
  было в шпорах; при необходимости добавь вручную.
